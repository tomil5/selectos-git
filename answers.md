## Вопросы заказчику перед тестированием

## О чем задача
Тестируется совместимость **Git** с **SELECTOS**.

---

## Что я бы запросил у заказчика

- Какую версию SELECTOS использовать?
- Какая архитектура у процессора?
- Какой пакетный менеджер использовать? (apt - по умолчанию?)
- Есть ли доступ в интернет?
- Есть ли внутренний репозиторий для установки (зеркала)?

- Какую версию Git тестируем? Какая минимально поддерживаемая?
- Нужно ли тестировать обновление между версиями?

- Как Git будет использоваться: только локально или с удалёнными репозиториями (GitHub)?
- Нужна ли поддержка SSH-ключей?
- Нужна ли поддержка Git LFS?

- Git будет запускаться от имени root или обычного пользователя?
- Нужна ли поддержка работы с несколькими пользователями на одной машине?
- Требуются ли ограничения через sudoers?

- Что является успешным результатом тестирования? (Git устанавливается и базовые команды работают? Или нужен больший охват?)
- Сущесвуют ли какие-то известные проблемы совместимости, которые нужно проверить в первую очередь?
- Нужно ли сравнивать поведение SelectOS с based системой? (Debian)

- Нужен ли отчет по проведенном тестировании?
- В каком формате предоставить результаты? (HTML-отчёт, PDF, таблица, .md и другие форматы)
- Куда сохранять логи тестирования? И сохранять ли вообще, может быть отобразить только в отчете?
